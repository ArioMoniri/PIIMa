deid-tr

Turkish clinical de-identification, entirely on this machine.

What this build removes, and what it leaves in

  • Removed: identifiers the L1 rule layer can prove - TCKN, VKN, IBAN, phone numbers, e-mail addresses, MRNs and dates, checksum-validated where the format carries a checksum.
  • NOT removed: names. Patient, clinician and relative names are the job of the L2 model ensemble. No L2 model is installed in this build and no weights ship with it, so deid-tr masks zero names. A name in your document is still in the output.
  • NOT removed: quasi-identifiers in prose - employer, relationships, assets, distinctive events. That is L3, which needs a local model you supply.
  • Not read: pixels. deid-tr has no OCR. A scanned page is refused rather than returned looking processed.

De-identify text

Nothing typed here leaves this machine. There is no network code in this application and its window is not allowed to make a request.

De-identify a file

PDF, DOCX, TXT, CSV, JSON, JSON Lines. The file is opened, rewritten and saved by the native side; its contents never enter this window and this window is never told what it was called.

Which layers are live

Read from the pipeline this process actually built, not from a configuration file.

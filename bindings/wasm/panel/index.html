deid-tr

Turkish clinical de-identification, entirely in this tab.

What this panel does and does not remove

  • Removed: identifiers the L1 rule layer can prove - TCKN, VKN, IBAN, phone numbers, e-mail addresses, dates and the other fixed-format direct identifiers, checksum-validated where the format carries a checksum.
  • NOT removed: names. Patient, clinician and relative names are the job of the L2 model ensemble. No L2 model is loaded in this build, so deid-tr masks zero names here. A name you paste stays in the output.
  • NOT removed: quasi-identifiers in prose - employer, relationships, assets, distinctive events. That is L3, which needs a local model this page does not have.
  • Paste synthetic text only. The tab never uploads anything, but the text is still in your clipboard and your browser process.

Extracting text from those formats needs a parser, and a redacted PDF must also have its text layer and its metadata scrubbed and then verified - work that does not belong in a page whose whole claim is that it ships no extra runtime. Policy

Raising this lowers recall. Every span below the threshold is left in the output. Invariant I2 of this project: a missed identifier is a breach, an over-mask is a papercut - so the two directions are not symmetric and this slider is only ever safe to move down. Checksum-validated spans sit at confidence 1.00 and are unaffected.

Redaction methods are a panel-side preview. Only surrogate is the real thing: it shows what the wasm module's L5 engine produced. Mask, redact, hash, date-shift and remove are applied here in JavaScript over the span map, because RedactionPolicy is not part of the wasm binding's exported API yet. When it is, these controls move onto it and the difference disappears. The panel-side hash is a non-cryptographic FNV-1a digest for display only - it is not the keyed HMAC a real hash method needs.

Starting.

        
        

          What is blocked. This page's Content-Security-Policy
          is default-src 'none' with
          img-src 'none' and
          font-src 'none';
          connect-src 'self' permits exactly one
          request, the module loading its own .wasm
          from this origin. The browser refuses everything else before it
          reaches the network, including image, media and font loads that no
          script can intercept.
        

        

          What is stubbed. Every networking global a script can
          name - fetch,
          XMLHttpRequest,
          WebSocket,
          EventSource,
          WebTransport,
          RTCPeerConnection,
          navigator.sendBeacon - was replaced with a
          throwing stub immediately after the module's own bytes were fetched
          and before any note could be read. These are function replacements, so
          they cover calls and nothing else: a declarative load such as
          new Image().src is not a call on a global
          and no stub can see it.
        

        

          What is measured. The counter above reads the
          browser's own resource timeline, which records every request the page
          actually made regardless of how it was made - so it covers the
          declarative loads the stubs cannot. Any non-zero reading turns the
          verdict red. Open devtools, switch to the Network tab and reload: you
          will see this document, the stylesheet, the scripts, the wasm-bindgen
          glue and the .wasm module, and nothing after that no matter what you
          paste.
        

      

    


    

      

        Safe Harbor here means L1 rules, span algebra, L4 routing and L5
        surrogates, compiled to WebAssembly from the same Rust the CLI runs. L2
        would need onnxruntime-web and a trained
        checkpoint; L3 would need a WebGPU-hosted local model. Neither is
        bundled, and the panel says so rather than degrading quietly.

deid-tr

Turkish clinical de-identification, entirely in this tab.

What this panel does and does not remove

  • Removed: identifiers the L1 rule layer can prove - TCKN, VKN, IBAN, phone numbers, e-mail addresses, dates and the other fixed-format direct identifiers, checksum-validated where the format carries a checksum.
  • NOT removed: names. Patient, clinician and relative names are the job of the L2 model ensemble. No L2 model is loaded in this build, so deid-tr masks zero names here. A name you paste stays in the output.
  • NOT removed: quasi-identifiers in prose - employer, relationships, assets, distinctive events. That is L3, which needs a local model this page does not have.
  • Paste synthetic text only. The tab never uploads anything, but the text is still in your clipboard and your browser process.

That page has text deid-tr can read and images it cannot. It redacts text and never touches pixels, so those images would have come back untouched inside a file that said "redacted" on it. A small image in a Turkish clinical report is very often a QR or barcode carrying the protokol or patient number; a wide short one is usually a signature or letterhead. deid-tr does not decode images and cannot tell you which this is.

Ticking this changes nothing about the images: they still travel into the output unread. It changes only whether you get the file, and every skipped image is then listed by page and pixel size beside the result. Drop the file again after ticking it.

The text on that page is pixels, not characters. Removing it needs OCR to find the words and an image editor to paint them out. deid-tr has neither and will not pretend to.     Raising this lowers recall. Every span below the threshold is left in the output.             What is blocked. This page's
            Content-Security-Policy is
            default-src 'none' with
            img-src 'none' and
            font-src 'none';
            connect-src 'self' permits exactly one
            request, the module loading its own .wasm
            from this origin. The browser refuses everything else before it
            reaches the network, including image, media and font loads that no
            script can intercept.
          
    
          
    
            What is stubbed. Every networking global a script
            can name - fetch,
            XMLHttpRequest,
            WebSocket,
            EventSource,
            WebTransport,
            RTCPeerConnection,
            navigator.sendBeacon - was replaced with a
            stub immediately after the module's own bytes were fetched and
            before any note could be read. The first six throw.
            navigator.sendBeacon records the call and
            returns false rather than throwing,
            because sendBeacon's contract is to return
            a boolean and a caller is entitled to read one; it still sends
            nothing and still increments the counter above. These are function
            replacements,
            so they cover calls and nothing else: a declarative load such as
            new Image().src is not a call on a global
            and no stub can see it.
          
    
          
    
            What is measured. The counter above reads the
            browser's own resource timeline, which records every request the
            page actually made regardless of how it was made - so it covers the
            declarative loads the stubs cannot. Any non-zero reading turns the
            verdict red. Open devtools, switch to the Network tab and reload:
            you will see this document, the stylesheet, the scripts, the
            wasm-bindgen glue and the .wasm module, and nothing after that no
            matter what you paste.
          
    
        
    
      
    
    
    

    
    
      
    
        Safe Harbor here means L1 rules, span algebra, L4 routing and L5
        surrogates, compiled to WebAssembly from the same Rust the CLI runs. L2
        would need onnxruntime-web and a trained
        checkpoint; L3 would need a WebGPU-hosted local model. Neither is
        bundled, and the panel says so rather than degrading quietly.
      
    
      
    
        The module also carries the container parsers - DEFLATE, PKZIP, the
        OOXML part sweep and an ISO 32000-1 PDF object model - which is why it
        weighs what the figure at the top of the page says it does. That figure
        is measured from the bytes this tab actually fetched, not written into
        the markup, so it cannot drift away from the build you are running. The
        parsers are the reason a PDF can be redacted here instead of being sent
        somewhere that has them.
